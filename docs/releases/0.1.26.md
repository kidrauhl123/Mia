# Mia 0.1.26

- 增加托管 MCP runtime 和 Xiaohongshu MCP 连接器，支持从 Mia 管理原生 MCP 模板、安装状态、健康检查和 agent 暴露。
- 强化 managed MCP supervisor，修复安装恢复、失败隔离、stale connected 状态、enable/sync gating 和 exposure fingerprint。
- 更新 MCP 连接库界面，隐藏旧的手动命令引导，改为连接向导和更清晰的托管连接状态。
- 修复 cloud Hermes 旧模型配置规整，并让 cloud agent dispatcher/model 路径兼容新的模型配置形态。
- 修复云端 Agent 流式输出相邻文本块被拆成多个气泡的问题，回复会按同一次文本流合并展示。
- 增加本地 Agent Doctor：Hermes 会检查 CLI 与 API 依赖，Claude Code、Codex、OpenClaw 会做启动自检；失败时设置页和首次启动引导会显示可修复状态与修复入口。
- 增加持久设备身份，避免 Core / bridge 运行期设备标识漂移。
- 修复移动端未读状态、前后台缓存同步、Android SQLite runtime 隔离，以及桌面聊天消息动效和 MCP 文本样式。
- 优化 trace 行动画、notes overflow、滚动条显示时机、聊天样式、创建智能体运行目标选择和 Windows 桌面 chrome surface。
- 扩展 Core MCP、managed service、renderer MCP library、cloud bridge/events/routing、bot runtime、移动端会话缓存和 Windows chrome 相关测试覆盖。
