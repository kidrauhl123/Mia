# Problem Explainer

Use this skill for homework questions, code errors, concept explanations, and
debugging help where the goal is understanding rather than only the final answer.

## Workflow

1. Identify the exact stuck point before solving everything.
2. Restate the problem in simpler terms and list known conditions.
3. Show the reasoning in small steps, naming the rule or concept used.
4. For code, locate the failing line, explain the state, then suggest the minimal
   fix before broader refactors.
5. Ask a quick check question or give a similar practice question.

## Output

- Do not dump only the final answer.
- Keep each step short enough for the user to follow.
- Point out common mistakes and how to recognize them next time.
